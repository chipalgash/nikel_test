# RAG-система для анализа технических документов

### Схема: 
Запрос → [Гибридный поиск] → [Reranker] → [LLM генерация] → [Валидация]
   ↓           ↓                 ↓            ↓              ↓
 Embedding  FAISS+BM25     Переранжирование  Структурированный  Автоматическая
           (20 чанков)     (топ-10 чанков)   JSON ответ        оценка качества

## 🎯 Финальное решение

### Архитектура системы:
1. **Предобработка**: Извлечение текста из DOCX, включая таблицы
2. **Чанкинг**: 900 символов с перекрытием 200
3. **Индексирование**: FAISS для семантики + BM25 для лексики
4. **Поиск**: Гибридный ансамбль с равными весами
5. **Переранжирование**: BGE reranker для точности
6. **Генерация**: Qwen3:14b с структурированным выводом
7. **Валидация**: Автоматическая оценка качества ответов

### Ключевые особенности:
- **Кэширование**: FAISS индексы сохраняются для быстрого перезапуска
- **Batch обработка**: Эффективная работа с reranker
- **Структурированный вывод**: JSON формат с ID использованных чанков
- **Автоматические источники**: Извлечение путей к документам
- **Детальное логирование**: Полная трассировка процесса

### Файловая структура:
```
app/
├── main.py           # Основной пайплайн
├── retriever.py      # Гибридный поиск + reranker
├── generator.py      # LLM генерация ответов  
├── models.py         # Модели данных (Chunk)
├── input_data.py     # Вопросы и эталонные ответы
└── requirements.txt  # Зависимости
└── result_answer.json  # Итоговые ответы модели

```

## 📊 Результаты экспериментов

### Что работает хорошо:
✅ **Гибридный поиск** - лучше чем отдельные подходы  
✅ **Переранжирование** - значительно улучшает релевантность  
✅ **Структурированный вывод** - надежное извлечение источников  
✅ **Кэширование** - быстрые повторные запуски  

### Узкие места и улучшения:
🔍 **Потенциальные улучшения:**
1. **Расширение контекста** - добавление соседних чанков для лучшей связности
2. **Метаданные чанков** - информация о разделах документов
3. **Настройка весов** - более точная балансировка BM25/FAISS
4. **Query expansion** - расширение запросов синонимами
5. **Постобработка** - улучшение форматирования ответов

⚠️ **Текущие ограничения:**
- Обработка только текстового контента (без изображений/схем)
- Фиксированные веса ансамбля (можно адаптировать по типу запроса)
- Отсутствие предварительной фильтрации по метаданным документов

## 🚀 Запуск системы

### Установка зависимостей:
```bash
pip install -r requirements.txt
```

### Запуск анализа:
```bash
# Полный пересчет
python main.py --rebuild

# Использование кэша
python main.py
```

### Результаты:
Система сохраняет результаты в папку `results/` с timestamp'ом, включая:
- Исходные вопросы
- Сгенерированные ответы
- Источники (ID чанков + пути к файлам)
- Оценки качества валидации
